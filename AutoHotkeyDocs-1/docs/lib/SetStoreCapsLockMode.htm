SetStoreCapsLockMode

Whether to restore the state of the CapsLock key after sending simulated keystrokes.

SetStoreCapsLockMode, OnOff

Parameters

OnOff

Specify one of the following values:

On: The CapsLock key will be restored to its former state if key-sending commands such as Send or ControlSend needed to change it temporarily for its operation.

Off: The state of the CapsLock key is not changed at all. As a result, key-sending commands will invert the case of the characters if the CapsLock key happens to be on during the operation.

[v1.1.30+]: The decimal values 1 and 0 may be used in place of On and Off, respectively.

Remarks

If SetStoreCapsLockMode is not used, the default setting is On. However, this does not guarantee that the CapsLock key will be turned off before sending keystrokes, nor that its previous state will be restored afterward.

SetStoreCapsLockMode is rarely used because the default behavior is best in most cases.

SetStoreCapsLockMode is ignored by Blind mode and in [v1.1.29+] by Text mode; that is, the state of the CapsLock key is not changed in those cases.

[v1.1.23+]: The built-in variable A_StoreCapsLockMode contains the current setting.

Every newly launched thread (such as a hotkey, custom menu item, or timed subroutine) starts off fresh with the default setting for this command. That default may be changed by using this command in the auto-execute section (top part of the script).

Examples

Causes the state of the CapsLock key not to be changed at all.

SetStoreCapsLockMode, Off
